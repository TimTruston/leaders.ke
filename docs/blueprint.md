### Basic Features

**Platform**
1. Seeds all (leadership) positions in the country by region, political party and alliances
2. States the official monthly compensation packages set by the **Salaries and Remuneration Commission (SRC)** for voters to get transparency
3. Automatically creates structured leader profiles for current leaders and existing party structures
4. Creates campaign pages and leader pages for every known candidate running in 2027 from public declarations and IEBC notices
5. Daily - aggregates news from verified media outlets, summarizes and and tags the mentioned leader profiles

**Campaign**
1. A public page with a clear civic mission: e.g. A candidate asking for votes, a public awareness mission, citizen mobilization...
2. A campaign can belong to another campaign (nested)
3. Do not cascade on delete
4. Broadcasts news to campaign subscribers (citizens)
5. Has an AI agent to respond to citizen queries

**Leader**
1. Sign up as a user (google oauth optional)
2. Creates or claims a profile and state their designation
3. Outline manifesto
4. Add Campaign Managers
5. Track news / PR

**Campaign Manager Roles (configurable)**
1. Add other managers
2. Remove other managers
3. Edit leader profile and manifesto
4. Create and manage a campaign
5. Upload and manage documents and media on a campaign or leader profile
6. Add ambassadors to a campaign
7. Confirm ambassador physical appearances
8. Manage news - create, edit, delete posts and tags
9. View analytics

**Ambassador Roles**
1. Add citizens to campaigns
2. Engage with citizen chats
3. Mobilize citizens for physical events
4. Appear in physical events
5. Collect citizen feedback

---
A leader record is just a join between a user and chair
1. Platform seeds:
	1. positions - known leadership positions
		- show current salary using SRC
	2. leaders
		- past and present and confirmed candidates
		- public page joining a user and chair
		- gets listed in user's page under "Track record"
	3. campaigns (confirmed campaigns)

2. Everyone signs up as a user - past and present leaders, candidates, managers, ambassadors, citizens
3. Any users can create campaigns but only accounts with verified campaigns get a public page (Leader Profile)
4. Candidate Journey - sign up, create a campaign or be added to one, pay subscription, get verified, go public, optionally add campaign managers 
5. Manager Journey - sign up, create a campaign or be added to one, pay subscription, get verified
6. Managers can sign up an create campaigns without needing the leader to sign up
7. Unverified managers and campaigns remain hidden
8. Ambassador Journey -  accept invitation from manager (requires sign up or log in), be connected to a campaign
9. Candidates who have never been leaders before don't have a leadership records in their public pages
10. Verified Candidate capabilities
	- assign roles to other users and thereby they become "managers"
		- edit candidate's public profile (leader record)
		- manage sub-campaigns - create, edit, delete, modify visibility
		- add/remove other managers
		- add ambassadors
	- accept/reject sub-campaigns
11. Campaigns
	- 1 main campaign per candidate
	- Nested campaigns accepted under main campaign
	- Main campaign has to be approved by admin and be paid for
	- Main campaign and sub campaigns are only visible if they have a paid, active subscription
	- Reject if campaign already exists, has been approved and paid for
12. Subscriptions 
	- belong to campaigns
	- only be paid for a saved campaign.
	- are paid for every single month or annually
	- active if endsAt is in the future
	- can be paid for by a candidate or manager

13. verification process
	- create a user account (candidate or campaign manager)
	- create a campaign
	- submit documentation for verification - id, proof of IEBC application, party pages, contacts
	- wait for approval - platform admin manually verify submission
	- pay a subscription
	- becoming verified
		- if the actual leader has not signed up, creates a user account for the leader, credentials saved
		- create leader page if it was not seeded
		- fills leaders.userId and leaders.verifiedAt
		- assigns leader_id to campaign and manager
		- fill campaign.verifiedAt, leader.verifiedAt, manager.verifiedAt

14. user can claim a leaders page - requires strict verification to prevent hacks
	- provide proof of subscription payment
	- proof association with prior creator
	- otherwise proof prior profile creator is false
